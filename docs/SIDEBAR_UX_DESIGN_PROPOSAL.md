# 侧边栏「项目 + 会话」交互设计方案（参考 ChatGPT）

在现有功能基础上，对齐 ChatGPT 的侧边栏体验：**项目可删除、会话移动用悬浮选择、整体更清晰**。

---

## 一、设计目标

| 目标 | 说明 |
|------|------|
| 项目可删除 | 每个项目行有**明确的删除入口**（独立删除按钮或 ⋮ 内删除），删除前二次确认 |
| 会话移动悬浮化 | 点击会话行 **⋮** 后，**「移动到项目」的目标列表**以**悬浮框**形式出现在 ⋮ 附近（类似 ChatGPT 的 popover），而不是底部 expander |
| 结构清晰 | 项目区、会话区层次分明；操作入口一致（⋮ = 菜单，🗑 = 删除） |

---

## 二、整体布局（从上到下）

```
┌─────────────────────────────────────────┐
│  对话记录                               │
│  [新建会话]                             │
├─────────────────────────────────────────┤
│  当前项目                               │
│  ● 全局会话                             │  ← 无菜单、无删除
│  ○ 项目A                          ⋮ 🗑  │  ← 选中高亮；⋮ 菜单；🗑 删除
│  ○ 项目B                          ⋮ 🗑  │
│  ○ Little P                       ⋮ 🗑  │
│  [+ 新建项目]                           │
├─────────────────────────────────────────┤
│  会话列表（当前项目下）                  │
│  03-02 11:14 | 单像素成像         ⋮ 🗑  │  ← ⋮ 点开 = 悬浮「移动到项目」
│  02-28 17:04 | 单像素成像         ⋮ 🗑  │
│  [▶ 展开更早会话] / [▼ 收起更早会话]    │
└─────────────────────────────────────────┘
```

## 三、项目区设计

### 3.1 列表形式（推荐，与现有 radio 对齐）

- **全局会话**：固定第一行，左侧 ● 表示选中；**无 ⋮、无 🗑**（不可删、不可重命名）。
- **项目行**：每行 = **单选圆点 + 项目名 + ⋮ + 🗑**
  - 点击整行或圆点 → 切换当前项目（与现有 `project_radio` 一致）。
  - **⋮**：点击展开**悬浮菜单**（Popover），项为「重命名」「删除」；删除走二次确认。
  - **🗑**：**直接删除入口**（与 ChatGPT 一致）；点击后可在同一行或 Popover 内做二次确认（「确定删除？其下会话将变为全局会话。」），确认后软删除并若当前选中的是该项目则切回「全局会话」。

### 3.2 删除按钮的两种实现方式

| 方式 | 优点 | 缺点 |
|------|------|------|
| **A. 每行独立 🗑** | 删除入口明显，符合「需要可以删除的按钮」 | 占横向空间，项目多时略挤 |
| **B. 仅 ⋮ 内「删除」** | 界面更简洁 | 删除多一步，不如 A 直观 |

**推荐**：**A**。项目数量通常不多，每行右侧 **⋮ + 🗑** 与下方会话行一致，认知统一；删除前用 `st.dialog` 或 Popover 内确认即可。

### 3.3 技术实现要点（项目区）

- 将当前 **`st.radio` 项目列表**改为**自定义行**：每行用 `columns` 或一层布局，左侧为可点击的「项目名」（或 radio 的视觉等价），右侧为 **⋮**（`st.popover`）+ **🗑**（`st.button`）。
- **⋮** 使用 **`st.popover("⋮")`**，内容为「重命名」「删除」；删除时在 Popover 内或 `st.dialog` 中二次确认。
- **🗑** 点击 → 直接打开**同一行的删除确认**（可用 `st.dialog` 或 Popover 内一段确认文案 + 确定/取消），确认后调用 `chat_store.delete_project(project_id)`，并清理 `project_id` / `project_radio`。

## 四、会话行设计（⋮ → 悬浮「移动到项目」）

### 4.1 当前问题

- 点击会话行 **⋮** 后，在侧边栏**底部**用 expander 展示「会话操作」和「移动到项目」列表。
- 列表与当前行分离，不符合 ChatGPT 的「点哪在哪」的悬浮选择体验。

### 4.2 目标交互（参考 ChatGPT）

- 用户点击某条会话的 **⋮** → 在该行 **⋮ 附近**出现一块**悬浮框**（Popover）。
- 悬浮框内容：
  - 标题/说明：如「移动会话到…」或会话标题缩写。
  - **移出项目**（若当前会话属于某项目）：点击后 `set_conversation_project(conv_id, None)`，关闭悬浮框并刷新列表。
  - **全局会话**：同上，语义与「移出项目」可合并或二选一展示。
  - **项目列表**：除当前所属项目外的所有项目，每项一行；点击某项目 → `set_conversation_project(conv_id, project_id)`，关闭悬浮框并刷新。
  - **关闭**：仅关闭悬浮框，不移动。

### 4.3 技术实现要点（会话行）

- **Streamlit 1.54** 支持 **`st.popover`**，适合做「附着在触发器旁的悬浮层」。
- 在 **`_render_history_rows`** 中，**第二列**（当前为 ⋮ 按钮）改为：
  - 使用 **`with st.popover("⋮", help=S["conv_move_to_project"]):`** 作为触发器；
  - Popover 内部根据当前行 `cid` 渲染：
    - 若 `conv_project_ids.get(cid)` 非空 → 显示「移出项目」按钮；
    - 遍历 `_projects`，排除当前所属项目，每个项目一个按钮；
    - 底部「关闭」按钮（或点击外部关闭）。
- 这样**每条会话行都有自己的 ⋮ Popover**，点击 ⋮ 后「选择目标项目」的列表以**悬浮框**形式出现在 ⋮ 旁，无需再在底部用 expander。
- 删除会话仍保留 **🗑**，逻辑不变（删除会话而非移动）。

### 4.4 悬浮框样式（可选）

- 已有 `theme_legacy` 中的 `.kb-conv-popover-panel`、`.kb-conv-popover-scroll` 可复用于「移动到项目」的 Popover 内容区（最大高度、圆角、阴影），使列表过长时可滚动，视觉与 ChatGPT 的浮层接近。

## 五、状态与边界（保持现有逻辑）

- **切换项目**：只改 `project_id`，会话列表切换；当前会话 `conv_id` 可不变，若不属于新项目则主区仍显示该会话，侧栏列表可置顶显示当前会话（已实现）。
- **删除项目**：软删除，其下会话 `project_id` 置为 NULL；若当前选中的是该项目，则 `project_id` 置为 None，切回「全局会话」。
- **移动会话**：`set_conversation_project` 后，若当前在「全局会话」视图且被移入某项目，列表可刷新后该会话从当前列表消失；若当前在该项目视图，被移出后同理。关闭 Popover 并 `st.rerun()` 即可。

## 六、实现优先级建议

| 优先级 | 内容 | 说明 |
|--------|------|------|
| **P0** | 会话 ⋮ 改为 Popover + 悬浮「移动到项目」列表 | 用 `st.popover` 替代底部 expander，每行 ⋮ 打开自己的浮层 |
| **P0** | 项目行增加删除入口 | 每项目行右侧 ⋮ + 🗑；🗑 或 ⋮ 内「删除」均可，删除前二次确认 |
| **P1** | 项目区改为「行 + ⋮ + 🗑」 | 用自定义行替代单一 `st.radio`，使每行有 ⋮ 和 🗑（或先 ⋮ 内删除再考虑独立 🗑） |
| **P2** | 悬浮框样式统一 | 复用/调整 `.kb-conv-popover-panel`，使「移动到项目」Popover 与现有主题一致 |

## 七、与 ChatGPT 的对应关系

| ChatGPT 行为 | 本方案 |
|--------------|--------|
| 项目列表，点击切换 | 保留「当前项目」单选列表，点击行切换 |
| 项目有菜单（重命名/删除） | 每项目行 ⋮ → Popover：重命名、删除 |
| 会话行 ⋮ → 移动会话 | 会话行 ⋮ → **Popover** 内「移出项目」+ 项目列表，选目标即移动 |
| 删除在菜单或图标 | 项目行 🗑 或 ⋮ 内删除；会话行 🗑 删除会话，⋮ 只负责移动 |

---

## 八、小结

1. **去掉外层强调、只留内层窄条** 的视觉已在前序修改中完成；本方案专注**交互**：项目可删、会话移动悬浮化。
2. **项目**：每行提供 **⋮ + 🗑**，删除有明确入口；⋮ 内重命名/删除，🗑 可直接删除（配合二次确认）。
3. **会话**：**⋮ 点击 → 悬浮框（Popover）**，内为「移出项目」+ 目标项目列表，选择即移动并关闭浮层，不再使用底部 expander。
4. 技术栈：**Streamlit 1.54** 的 **`st.popover`** 实现会话移动的悬浮选择；项目区用自定义行 + `st.popover` + `st.button`（及可选 `st.dialog` 做删除确认）即可落地。
