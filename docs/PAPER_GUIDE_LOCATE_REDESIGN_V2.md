# 文献阅读指导定位功能改版实施方案（V2）

## 1. 问题定义

当前定位体验的核心问题不是“是否能跳转”，而是“入口表达与定位质量不可控”：

1. 入口位置不贴合阅读流，用户不容易在句子旁就近触发。
2. 定位点数量偏多，弱相关片段也在显示入口，造成视觉噪声。
3. 前端存在较多事后猜测匹配，弱化了“证据可追溯”的可信度。
4. 公式/长段落场景命中不稳定，用户感知为“跳得不准”。

目标是把“定位”从附加按钮改为“答案的一部分”，并且默认可信、可解释。

## 2. 产品目标与边界

### 2.1 产品目标

1. 入口必须就近：放在对应句子末尾的内联微标记（下引号样式）。
2. 入口必须克制：每条回答最多 1-3 个定位入口，只在高价值句出现。
3. 跳转必须可解释：点击后告诉用户命中的是哪段证据，若降级也要提示。
4. 质量必须可量化：有明确命中率、可见率、回退率指标。

### 2.2 非目标（本轮不做）

1. 不做跨多篇文献联合证据图谱。
2. 不重做整套 Reader 视觉系统。
3. 不追求历史旧消息全部补齐高精定位。

## 3. 设计原则

1. 证据优先：没有足够证据就不显示入口。
2. 后端产证据，前端做表达：前端尽量不做重匹配推断。
3. 少而准：宁可少显示，不做“满屏定位”。
4. 明确降级：从精确到模糊到 LLM 修复，每一步可观测。

## 4. 目标体验（UX）

### 4.1 入口形态

1. 主入口：句尾内联微标记（`❞`），默认可见、低对比。
2. 交互方式：点击即定位；悬停仅增强视觉，不作为必须条件。
3. 可点区域：视觉小、命中框不小于 18x18。
4. 可访问性：支持键盘聚焦与 Enter 触发。

### 4.2 信息密度控制

1. 每条回答入口总数 `1-3`。
2. 每个逻辑区块（结论/依据/下一步）最多 1 个（后续通过 segment 分区实现）。
3. 列表逐项、公式逐行默认不打点，优先关键断言句。

### 4.3 点击反馈

1. Reader 平滑滚动到目标块并高亮。
2. 顶部/侧边显示“命中来源：章节 + 块类型”。
3. 若降级命中，提示“未精确命中，已跳到最相关段落”。

## 5. 后端与 LLM 方案

### 5.1 证据结构输出（持续沿用并加强）

每条 assistant 消息携带 provenance：

1. `segments[]`：回答段片。
2. `evidence_block_ids[]`：每段证据块。
3. `evidence_confidence`：映射置信度。
4. `mapping_mode`：`fast` / `llm_refined`。
5. `llm_rerank_calls`：LLM 修正调用次数。

### 5.2 LLM 在定位中的职责

1. 生成阶段：对低置信候选做 rerank，减少错配。
2. 点击阶段：当精确锚点失效时，LLM 在候选块中做“最可能块”重判。
3. 约束：LLM 只做候选内选择，不允许无约束生成跳转目标。

### 5.3 定位修复链路

1. 一级：`block_id + anchor_id` 精确命中。
2. 二级：同文档 fuzzy（标题路径 + 文本重叠）。
3. 三级：LLM 修复（点击句 + TopK 候选块）。
4. 兜底：明确提示当前为降级定位结果。

## 6. 前端改造方案

### 6.1 渲染策略

1. 定位入口只在段落主句（`p`）末尾渲染。
2. 默认不在 `li` 和 display math 节点逐项渲染，避免泛滥。
3. 去除右侧固定/绝对定位按钮样式。
4. 保留底部摘要，不再列出大批可点入口。

### 6.2 数量与优先级控制

1. structured 模式仅取 Top 3 证据段。
2. 优先高置信段；低置信段不显示入口。
3. 同一 segment 只渲染一次入口，避免重复。

### 6.3 状态与可观测性

1. 前端记录：入口渲染数、点击数、命中/降级命中。
2. 后端日志记录：`mapping_mode`、`llm_rerank_calls`、fallback 次数。

## 7. 分阶段实施

### P0（立即）UI 与密度止血

1. 入口改为句尾内联 `❞`。
2. 去掉右侧悬浮胶囊按钮。
3. 入口总量限制 1-3。
4. 禁止列表/公式逐项出入口。

验收：用户在回答正文旁能直接看到入口，视觉不突兀。

### P1（本周）结构化选点收敛

1. structured entry 默认 `maxEntries=3`。
2. 引入置信度阈值筛选低价值段。
3. 同 segment 去重。

验收：平均每条回答定位入口 <= 2.2，且关键结论句覆盖稳定。

### P2（本周）点击修复增强

1. 强化 Reader 的二级 fuzzy 匹配。
2. 引入点击时 LLM 修复（受开关控制）。
3. 输出“命中级别”给 UI 展示。

验收：首次点击命中率 > 90%。

### P3（后续）分区限额与学习式优化

1. 把回答 segment 绑定到结论/依据/下一步分区。
2. 每分区最多 1 个入口。
3. 使用线上点击反馈优化选点策略。

验收：入口减少但点击成功率持续上升。

## 8. 测试与验收

### 8.1 功能测试

1. 有 direct evidence 的回答应出现入口。
2. 仅 synthesis 段不应出现入口。
3. 点击后能打开绑定文献并高亮块。

### 8.2 回归测试

1. 普通聊天模式不受影响。
2. 旧消息无 provenance 时不崩溃。
3. 公式问答不会退化为全失败定位。

### 8.3 指标门槛

1. 入口可见率（有证据时）> 98%。
2. 首次点击命中率 > 90%。
3. 降级定位率 < 12%。

## 9. 实施清单（当前迭代）

1. 新入口样式与渲染位置改造（前端）。
2. structured 入口数量与置信度阈值控制（前端消费层）。
3. 维持现有 async LLM rerank，并补点击修复埋点（后端/前端）。
4. 构建与单测回归，验证端口服务可用。

## 10. 风险与应对

1. 风险：入口减少后用户担心“是不是没证据”。
2. 应对：保留摘要提示“本回答有 N 处可定位证据”。
3. 风险：低置信阈值过高导致关键点漏出。
4. 应对：阈值配置化并基于日志回放调优。
