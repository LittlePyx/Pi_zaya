## 1. 目标与边界

### 1.1 目标

在不牺牲响应速度的前提下，提升会话回答的**可读性、可追溯性、稳定性**，减少“答非所问/引用弱绑定/用户不知道下一步怎么读”的体验问题。

### 1.2 本轮明确做

1. 回答结构统一（结论优先 + 证据支撑 + 不确定性声明 + 下一步建议）。
2. 引用锚定增强（命中库内片段时，回答与引用绑定更稳定）。
3. “无命中”场景降级策略优化（不胡编，给出可执行补充建议）。
4. 回答后处理规范化（展示 markdown、复制 markdown/plain text 一致）。
5. 质量评测与灰度开关（可回退）。

### 1.3 本轮不做

1. 领域阅读路径生成（后续“领域内阅读指导”阶段）。
2. 前沿推荐算法（后续“前沿阅读推荐”阶段）。
3. 完整会话内阅读器重构（保持现有链路，仅对回答层优化）。

### 1.4 目标用户画像（本计划重点）

1. `理工科硕士生`：高频需求是“读懂论文 + 快速复现 + 找实验切入点”。
2. `博士生`：高频需求是“方法比较 + 创新点抽象 + 可行性判断 + 研究设计”。
3. `高校教师/青年PI`：高频需求是“方向判断 + 课程/组会讲解 + 选题与前沿判断”。
4. 覆盖学科：数学、物理、化学、材料、电子、电气、机械、土木、计算机、交叉工程等。

### 1.5 典型问题类型（按科研工作流）

1. `文献阅读型`：这篇论文核心贡献是什么？公式/图如何理解？
2. `对比评估型`：A 方法和 B 方法差异、优劣和适用边界？
3. `想法探索型`：这个 idea 是否可行？创新性在哪里？风险是什么？
4. `实验设计型`：最小可验证实验怎么做？指标和对照如何设？
5. `复现排障型`：为什么复现不出来？应先排查哪里？
6. `写作表达型`：related work/方法描述如何组织更清晰？

说明：回答优化必须针对以上类型输出不同的“结构与深度”，不能一套模板打天下。

## 2. 成功标准（DoD）

### 2.1 线上行为标准

1. 结构完整率 >= 95%（有结论/证据/下一步建议三个核心块）。
2. 命中知识库时，证据引用覆盖率 >= 90%（关键结论可在 refs 中追溯）。
3. “未命中知识库”误报率显著下降（有 hits 时不再展示 miss notice）。
4. 回答可读性主观评分提升（内部评审问卷平均 >= 4/5）。
5. 想法探索型问题中，“可行性+风险+最小验证路径”三要素覆盖率 >= 90%。
6. 文献阅读型问题中，“结论可追溯到命中文段”的人工复核通过率 >= 95%。

## 4. 方案设计

### 4.1 回答协议（Answer Contract v1）

统一回答模板（模型侧约束 + 渲染侧兜底）：

1. `结论`：先给直接答案，1-3 句。
2. `依据`：列出来自命中片段的核心证据（带 `[n]` 或锚点）。
3. `边界`：说明不确定点/前提条件（无则省略）。
4. `下一步`：给 1-3 条可执行动作（例如“优先看某 section/某 figure”）。

说明：

1. 对短问答允许自动压缩版（不机械输出小标题）。
2. 对事实敏感问题强制保留证据段。

#### 4.1.1 分场景回答形态（针对科研用户）

1. `文献阅读型`
   - 输出顺序：一句话结论 -> 关键证据（引用）-> 术语解释 -> 建议先读位置。
2. `对比评估型`
   - 输出顺序：对比表（核心维度）-> 适用条件 -> 失败场景 -> 推荐选择策略。
3. `想法探索型`
   - 输出顺序：可行性判断 -> 可能创新点 -> 主要风险 -> 最小验证实验（MVP）-> 下一步文献建议。
4. `实验设计型`
   - 输出顺序：研究假设 -> 实验变量/对照 -> 指标与统计 -> 预期结果与判据 -> 常见陷阱。
5. `复现排障型`
   - 输出顺序：最可能原因（按优先级）-> 快速诊断步骤 -> 修复动作 -> 复验标准。
6. `写作表达型`
   - 输出顺序：结构建议 -> 可直接替换范句 -> 修改理由 -> 避免问题（冗长/逻辑跳跃）。

#### 4.1.2 深度档位（自动 + 可控）

1. `L1 简洁`：面向快速问答，5-8 行。
2. `L2 标准`：默认档，结论+证据+下一步。
3. `L3 深入`：面向博士/教师，增加假设边界、替代路径、实验建议。

策略：

1. 默认根据问题长度、术语密度、是否包含“idea/实验/对比”等意图词自动选档。
2. 后续可在 UI 增加“回答深度”开关（本轮先保留自动策略）。

### 4.2 后端改造点

1. `kb/task_runtime.py`
   - 抽出回答构建函数（建议新增 `build_answer_contract` 逻辑）。
   - 按 `has_hits/anchor_grounded` 区分 prompt 约束强度。
   - 增加轻量意图识别（阅读/对比/idea/实验/排障/写作）并路由到对应回答形态。
   - 优化 miss fallback 文案与“下一步建议”生成。

   - 增加 `answer_depth_auto`（自动深度）和 `answer_mode_hint`（可选提示）预留位。

## 7. 风险与回滚

### 7.1 风险

1. 结构约束过强导致回答冗长。
2. 证据约束过强导致模型流式首字变慢。
3. 清洗规则过严误删有用内容。
4. 跨学科问法语义差异大，单一规则易误判意图。

### 7.2 应对

1. 分问句类型启用不同约束强度（短问答自动压缩）。
2. 监控 `gen.answer` 时延，超阈值自动降级模板复杂度。
3. 所有关键行为加 feature flag，可秒级回滚。
4. 意图识别先“保守路由”，不确定时回退标准模板，避免错误强化。

### 7.3 回滚策略

1. 关闭 `answer_contract_v1`，恢复旧 prompt/后处理路径。
2. 保留日志与评测结果，下一轮按失败样本修正后再灰度。

   - 证据可追溯率
   - 可执行建议评分
