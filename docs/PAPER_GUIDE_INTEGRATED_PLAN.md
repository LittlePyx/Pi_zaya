# 文献阅读指导功能整合计划（Paper Guide）

更新时间：2026-03-09  
状态：`proposed`  
范围：`kb_chat` 当前主线增量，不重做历史架构

## 1. 背景与约束

基于当前两个计划文档：
- [ANSWER_OPTIMIZATION_PLAN.md](./ANSWER_OPTIMIZATION_PLAN.md)
- [CITE_SHELF_OPTIMIZATION_IDEAS.md](./CITE_SHELF_OPTIMIZATION_IDEAS.md)

本次新增能力需遵循以下约束：
1. 已有评测结论作为基线，不再跑大规模测试（20/37 条）。
2. 仅做最小必要验证：`health + quality_summary + 小样本点测`。
3. 优先延续未完成项：`D4 交互式手工回归` + 文献篮 `P1/P2` 可用性整理。
4. 新功能以“最小可用闭环”为目标，不做整站重构。

### 1.1 现状补充（2026-03-09 复盘）

1. 已确认阅读指导错文与 404 问题属于“运行版本落后于代码”的高风险项：日志出现 `guide_source` / `conversations/{id}/guide` 404，需先完成后端热更新（重启到最新代码）再继续验收功能行为。
2. 已确认问答慢点主因不在检索，而在 refs 富化链路：`refs_enrich/answer_refs_enrich` 常见 28-35s，导致用户体感“refs 阶段过长”。
3. 因此本计划优先级补充为：先做运行一致性与 refs 异步解耦，再推进阅读指导体验细化。

## 2. 目标与非目标

### 2.1 目标

实现“围绕单篇文献”的阅读指导闭环：
1. 新建专门会话并绑定一篇文献。
2. 会话中提到的文献位置可点击打开应用内 Markdown->HTML 阅读器。
3. 阅读器可定位并高亮指定位置（优先章节/锚点，兜底文本片段）。
4. 阅读器中选中文本可回写会话输入框，用户可追加提问。

### 2.2 非目标（本阶段不做）

1. 不做全新的检索引擎或完整阅读器重写。
2. 不做项目级协作/分享能力（仍按现有会话维度）。
3. 不做跨文献自动阅读路径生成（后续阶段再做）。

## 3. 与现有计划整合

### 3.1 A0（会话回答优化）未完成项整合

先完成：
1. `Week2 D4` 交互式手工回归补齐（使用现有脚本与清单）。
2. 灰度发布预案中的时延闸门样本补齐，形成最终 go/no-go 记录。

再进入 Paper Guide 功能开发，避免“质量回归”和“新功能问题”耦合。

### 3.2 文献篮 P1/P2 整合

在已有能力基础上，优先补“阅读闭环相关”操作：
1. 从引用弹层/文献篮进入“围绕本文”流程：  
   - 已入库（有本地 `md/pdf`）-> 一键开启阅读指导会话。  
   - 未入库（仅元数据）-> 先进入“准备本文”流程（上传/导入 PDF -> 转换入库），完成后再开启会话。
2. 文献篮条目“在阅读器打开并定位”仅对已入库条目启用；未入库时展示明确引导文案。
3. 弱化对用户价值低的内部追踪字段展示（保留调试能力，不前台暴露噪音）。

## 4. 优先级（执行顺序）

1. `P0.1` 运行一致性：后端切换到最新代码（消除 `guide_source/guide` 404），确认阅读指导绑定字段可读写。
2. `P0.2` refs 提速止血：确保“答案先返回、refs 后补全”，避免 `refs` 阻塞每轮问答完成。
3. `P1` MVP：单篇文献绑定会话 + 引用点击打开阅读器 + 定位高亮。
4. `P2` 闭环：阅读器选中回写会话输入框（追加提问）。
5. `P3` 体验整理：文献篮/引用入口统一、信息降噪、交互打磨。

## 5. 方案设计（MVP）

### 5.1 数据与会话模型

在 `conversation` 维度新增最小字段（可通过迁移补列）：
1. `mode`: `normal | paper_guide`
2. `bound_source_path`: 绑定文献路径（优先 md 路径，回退 pdf）
3. `bound_source_name`: 绑定文献展示名
4. `bound_source_ready`: 是否已入库可检索（未入库会话不进入强约束检索）

### 5.2 后端能力

1. 会话接口扩展
- 创建会话时可指定 `mode/bound_source_path/bound_source_name`。
- 提供读取/更新绑定信息接口（用于前端切换入口）。

2. 生成链路约束
- `paper_guide` 会话下，对检索结果按 `bound_source_path` 强约束（至少优先过滤到单文献）。
- 若无结果，明确返回“当前文献未覆盖”的提示，并给“扩展全库”建议。

3. 阅读器数据接口
- 新增“读取并渲染 Markdown HTML”接口（返回：`html + anchors + source meta`）。
- 支持定位参数：`heading_path` / `anchor_target_kind+number` / `snippet`。
- 提供高亮片段所需上下文（前端负责样式与滚动）。

### 5.3 前端能力

1. 会话入口
- 在引用弹层和文献篮条目增加“围绕本文”入口，并按条目状态分流：
  - 已入库：直接创建 `paper_guide` 会话。
  - 未入库：跳转“准备本文”流程，流程完成后自动回到创建会话。
- 会话头部显示“当前绑定文献”状态条。

2. 阅读器形态
- 先采用 Chat 页右侧抽屉（不跳离会话，不打断时间线）。
- 点击会话中的引用 chip 后，打开抽屉并自动定位高亮。

3. 选中回问
- 阅读器监听文本选择，出现“追加到提问”按钮。
- 将选中文本以引用块形式写入 ChatInput 草稿，用户可继续编辑再发送。

## 6. 分阶段排期（建议）

### 阶段 A（0.5-1 天）
1. 完成 D4 交互式手工回归。
2. 补齐时延闸门样本记录。

### 阶段 B（1-2 天，MVP 主链路）
1. 会话模型扩展与 API 改造。
2. `paper_guide` 检索约束接入。
3. 引用点击打开阅读器 + 定位高亮打通。

### 阶段 C（1 天，闭环）
1. 阅读器选中 -> 追加到提问草稿。
2. 草稿模板与来源标记统一。

### 阶段 D（1 天，体验整理）
1. 文献篮/引用入口统一与文案降噪。
2. 快速可用性修补（滚动、焦点、抽屉状态保持）。

## 7. 验收与最小验证

仅执行以下验证组合：
1. `health`：服务可用。
2. `quality_summary`：结构完整率/证据覆盖率无明显回退。
3. 小样本点测（建议 6-8 条）：
   - `guide_source` 与 `conversations/{id}/guide` 不再 404（运行版本一致）。
   - 单轮提问中 `status=done` 不再被 refs 富化长时间阻塞（观察 stage 与耗时日志）。
   - 创建绑定文献会话成功。
   - 未入库条目触发“准备本文”流程，不能误进阅读器/强约束问答。
   - 在会话中提问，回答引用可打开阅读器。
   - 阅读器定位与高亮有效。
   - 选中文本可回写输入框并成功追问。
   - 无命中时降级文案正确（不胡编）。

## 8. 风险与回滚

1. 风险：强约束检索导致“答不出”比例上升。  
回滚：增加“文献内优先，必要时扩展全库”的开关策略。

2. 风险：阅读器渲染/高亮影响页面交互（卡滚动、抽屉状态丢失）。  
回滚：保留当前引用弹层，不自动打开阅读器。

3. 风险：选中回写造成误触与草稿污染。  
回滚：改为二次确认（仅插入临时引用，不自动拼接问题句）。

## 9. 本轮交付定义（DoD）

1. 可创建并识别 `paper_guide` 会话。
2. 会话内引用可打开应用内阅读器并完成定位高亮。
3. 阅读器选中内容可回写会话输入框追加提问。
4. A0 未完成收尾项（D4 + 时延闸门样本）完成记录齐全。
5. 仅使用最小验证链路，无大规模评测回归。

## 10. 页面与信息架构设计（定稿）

### 10.1 是否新建页面

结论：**P1/P2 不新建独立页面**，在现有 Chat 页面内做“阅读指导模式”。

原因：
1. 复用现有会话、时间线、引用渲染和评测链路，改造成本最低。
2. 避免普通会话与阅读指导会话数据割裂。
3. 降低切页带来的状态丢失风险（滚动、时间线、输入草稿）。

补充：P3 可选增加“阅读器全屏模式”，但仍挂在同一会话路由下，不做独立产品线。

### 10.2 在哪里区分普通会话 vs 阅读指导会话

区分粒度：**会话级模式**（`conversation.mode`）。

区分位置：
1. 侧边栏会话项：增加模式标识（如 `阅读指导` 标签）。
2. 会话头部：展示“当前绑定文献卡片”（标题、入库状态、快捷动作）。
3. 消息区布局：`paper_guide` 默认打开阅读器联动区；`normal` 维持当前布局。

### 10.3 三个入口统一设计

1. 文献管理页入口
- 已入库：按钮 `开始阅读指导` -> 直接创建 `paper_guide` 会话并绑定文献。
- 未入库：按钮 `准备后开始` -> 先走上传/转换，完成后自动进入阅读指导会话。

2. 会话内上传入口
- PDF 入库完成状态条出现 CTA：`围绕这篇开始阅读指导`。
- 点击后：当前会话可切换为 `paper_guide`（或新建一个绑定会话，默认推荐新建）。

3. 参考定位入口（引用弹层 / RefsPanel）
- 新增按钮：`围绕此文阅读`。
- 若来源文献未入库则走“准备本文”流程；已入库则直接进绑定会话并打开定位。

## 11. 交互布局设计（桌面/移动）

### 11.1 桌面端（默认）

采用三段式但保持单页面：
1. 左：现有会话列表（不变）。
2. 中：会话消息区 + 时间线（保留）。
3. 右：阅读器侧栏（在 `paper_guide` 默认展开，普通会话默认收起）。

### 11.2 移动端（默认）

不做三栏，改为分层：
1. 主层：消息区。
2. 阅读器：底部抽屉/全屏抽屉。
3. 选中文本后浮动按钮：`追加到提问`。

### 11.3 关键联动

1. 点击引用 -> 右侧阅读器打开并定位高亮，消息区保持可滚动。
2. 阅读器选中 -> 写入输入框草稿，不自动发送。
3. 时间线跳转后阅读器不消失，仅更新高亮定位。

## 12. 会话状态机设计

### 12.1 状态定义

1. `normal`
2. `paper_guide_pending`（绑定目标文献但未入库）
3. `paper_guide_ready`（绑定且可检索）

### 12.2 转换规则

1. `normal -> paper_guide_pending`：用户从任一入口选择了未入库文献。
2. `paper_guide_pending -> paper_guide_ready`：入库与转换完成。
3. `paper_guide_ready -> normal`：用户手动退出阅读指导模式（保留绑定信息可恢复）。

### 12.3 约束规则

1. `pending` 状态禁用“文献内强约束问答”和阅读器定位功能。
2. `ready` 状态启用“文献内强约束优先”，并允许一键扩展全库。

## 14. 可用性红线（必须满足）

1. 任意入口进入阅读指导后，用户始终能看到“当前绑定文献是谁”。
2. 点击定位后不允许出现“页面卡住不可滚动”。

4. 未入库文献不允许进入“假可用”状态（必须有明确引导）。
