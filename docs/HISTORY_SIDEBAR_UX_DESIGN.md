# 历史会话栏交互设计

目标：让「对话记录」整块（新建会话 + 项目 + 会话列表）结构清晰、操作有逻辑、符合直觉，并补全项目删除/重命名等能力。

---

## 一、整体结构（从上到下）

```
┌─────────────────────────────────────┐
│  [新建会话]   ← 主操作，始终在最上    │
├─────────────────────────────────────┤
│  对话记录                            │
│  ┌─ 项目 ─────────────────────────┐ │
│  │ ● 全局会话                      │ │  ← 固定第一项，无菜单
│  │ ○ 项目A                    ⋮   │ │  ← 选中项高亮，右侧菜单
│  │ ○ 项目B                    ⋮   │ │
│  │ [+ 新建项目]                    │ │  ← 最后一项，点击展开/弹输入
│  └────────────────────────────────┘ │
│  ─────────────────────────────────  │
│  会话列表（仅当前项目下的会话）       │
│  · 03-02 11:14 | 单像素成像    ⋮ 🗑 │ │  ← 每行：标题 + 菜单 + 删除
│  · 02-28 17:04 | 单像素成像    ⋮ 🗑 │ │
│  · ...                              │ │
│  [▶ 展开更早会话] / [▼ 收起更早会话]  │
└─────────────────────────────────────┘
```

**设计要点：**

- **新建会话**：永远在顶部，不随项目折叠；点一次 = 在当前选中的项目（或全局）下新建一条会话并切过去。
- **项目区**：用「单选列表」代替下拉框，一眼看到「全局 + 所有项目 + 新建项目」，点击行切换；当前项高亮（●/底色）。
- **会话列表**：只显示「当前选中的项目」下的会话；当前打开的会话置顶并高亮。

---

## 二、项目区交互

### 2.1 列表形式（推荐）

- **全局会话**：固定第一行，无右侧菜单（不能删/不能重命名）。
- **项目行**：每行 = 项目名 + 右侧 **⋮** 菜单（仅悬停或点击时出现，避免拥挤）。
- **⋮ 菜单项**：
  - **重命名项目**：弹出输入框（或行内 inline 编辑），确认后更新名称。
  - **删除项目**：二次确认（「确定删除该项目？其下会话将变为全局会话。」）→ 软删除（会话归到全局），若当前选中的正是该项目，则自动切回「全局会话」。
- **+ 新建项目**：单独一行在列表最下方；点击后：
  - **方案 A**：展开本行下方的输入框（项目名称 +「创建」按钮），创建成功后收拢并选中新项目。
  - **方案 B**：弹出 `st.dialog` 输入名称，确认后创建并选中新项目。

### 2.2 与现有下拉的取舍

- 若保留「当前项目」下拉：可在下拉右侧加一个 **⋮**，仅当选中「某项目」（非全局）时显示，点开为「重命名 / 删除」；「新建项目」仍用 expander 或 dialog。这样改动小，但删除/重命名入口略隐蔽。
- **推荐**：改为「单选列表 + 每行 ⋮」，项目数量通常不多，列表更直观、操作更一致。

---

## 三、会话行交互

### 3.1 每行内容

- **左侧**：会话标题（日期时间 | 标题摘要），点击 = 切换当前会话（与现有一致）。
- **右侧**：
  - **⋮**：会话菜单（见下）。
  - **🗑**：删除该会话（与现有一致）；删除后若删的是当前会话，则自动切到同列表下一条或新建一条。

### 3.2 会话菜单（⋮）

- **移动到项目**：子菜单列出所有项目（不含当前所属），选一个则 `set_conversation_project(conv_id, project_id)`；若当前选中的就是该项目，列表会立即更新（该会话从当前列表消失或出现）。
- **移出项目**：仅当该会话属于某项目时显示；点击后 `set_conversation_project(conv_id, None)`，会话变为全局会话。若当前在「全局会话」视图，列表会多出这一条；若当前在项目视图，该会话从列表消失。
- 可选：**重命名会话**（若后续支持修改 title）。

逻辑：先有「删除」，再有「移动/移出」，避免误删、方便整理。

---

## 四、状态与边界

### 4.1 当前项目 + 当前会话

- 切换项目：只改「当前项目」，会话列表立即切换为该项目的会话；**当前会话**（主区显示的那条）不变，若它不属于新项目，仍可在主区继续看，侧栏列表里用「置顶 + 补标题」显示（已实现）。
- 删除当前所在项目：确认后执行软删除，`project_id` 置为 `None`，侧栏自动切到「全局会话」，当前会话不变（因为会话还在，只是归到全局）。

### 4.2 空状态

- 当前项目下没有会话：列表区显示「暂无会话」，可提供「新建会话」快捷入口（或依赖顶部「新建会话」）。
- 没有任何项目：项目区只有「全局会话」和「+ 新建项目」。

### 4.3 新建会话

- 始终在「当前选中的项目」下新建（全局 = 当前项目为「全局会话」）；新建后自动选中该会话并置顶显示。

## 五、实现优先级建议

| 优先级 | 内容 | 说明 |
|--------|------|------|
| P0 | 项目删除 | 项目 ⋮ → 删除 → 二次确认 → 软删除；若删的是当前项目则切回全局 |
| P0 | 项目重命名 | 项目 ⋮ → 重命名 → 输入新名称确认 |
| P1 | 项目区改为列表 | 用单选列表替代下拉框，每项目行带 ⋮，底部「+ 新建项目」 |
| P1 | 新建项目入口收敛 | 列表最下「+ 新建项目」点击后 expander 或 dialog 输入名称 |
| P2 | 会话 ⋮ 菜单 | 「移动到项目」「移出项目」 |
| P2 | 空状态文案 | 当前项目无会话时显示「暂无会话」 |

---

## 六、文案与 key 建议

- 项目菜单：`project_rename`（重命名项目）、`project_delete`（删除项目）、`project_delete_confirm`（确定删除该项目？其下会话将变为全局会话。）
- 会话菜单：`conv_move_to_project`（移动到项目）、`conv_remove_from_project`（移出项目）
- 空状态：`no_conversations_in_project`（当前项目暂无会话）

以上为「历史会话栏」的完整交互设计，确认后可按 P0 → P1 → P2 在现有侧栏代码里落地。
